01Problem Space
Capability demonstrated → Problem framing & spatial systems thinking

The information integration problem

Bioregional planning in BC requires integrating diverse spatial datasets across multiple provincial repositories — terrain, ecology, resource tenures, cultural context, planning overlays — and synthesizing them into a coherent picture for each site under review. Without a structured system, that synthesis is slow, inconsistent, and nearly impossible to repeat at scale.

The core challenge: datasets exist across BC Data Catalogue, TANTALIS, federal sources, and field surveys — no standardized synthesis across sites, no consistent cartographic logic, no reusable production path. Every site analysis starts from scratch.

To address this, a scalable GIS architecture was designed to integrate provincial datasets, LiDAR-derived terrain analysis, and automated atlas production into a single repeatable framework. The result is not a collection of maps — it is a spatial information infrastructure.

Scale of the problem

20+ sites × 15+ themes means 300+ individual map products needed — all requiring consistent data, symbology, and layout logic. That scale is only tractable through systems design, not manual cartography.

02System Architecture
Capability demonstrated → Data abstraction layer design & BC ecosystem literacy

Three-layer data architecture

The framework is built on a three-layer architecture: source data → thematic data library → atlas production. The library acts as the abstraction layer — isolating the production system from raw source variation, and making it possible to update a dataset or add a new site without touching the cartographic pipeline.

03Production System
Capability demonstrated → GIS production pipeline design & repeatability

Designed for repeatability and modularity

A repeatable GIS production workflow was designed to integrate provincial datasets, LiDAR analysis, and automated atlas generation from a single data library. Adding a new study site requires only updating the data library — the entire production pipeline re-runs unchanged.

ArcGIS Pro Map Series (Data Driven Pages) drove cartographic output — once the template and symbology were configured, the same layout engine generated maps for every site automatically.

The pipeline was built modularly: LiDAR terrain analysis is one independent module, cartographic production is another. Each can be updated or re-run without disrupting the rest.

04LiDAR Automation
Capability demonstrated → Computational GIS & automated terrain analysis

Automating the hydrology theme

The hydrology and terrain theme required surface-derived products (DTM, DSM, CHM, slope, aspect, hillshade) and hydrological analysis outputs — all generated from raw LiDAR point clouds via an automated R + Python pipeline.

An R script using lidR handled point cloud normalization and surface derivation. A Python script ran hydrological analysis and fed results into a folium map for inline QA visualization — so outputs could be verified without opening GIS software between runs.

Why this matters at scale

Manual terrain and hydrology analysis for 20+ sites would have taken weeks. Automating the pipeline reduced per-site processing to a single script execution — and the embedded visual QA caught classification errors before they propagated into the atlas.

06System Capability
Capability demonstrated → Spatial information infrastructure design

What the framework delivers

The project established a scalable framework for synthesizing environmental, cultural, and resource datasets into a consistent bioregional atlas — turning a weeks-long manual process into a reproducible spatial intelligence product.

The deliverable is not a set of maps. It is a spatial information infrastructure — a system that integrates 40+ provincial datasets across environmental, cultural, and economic domains, combines LiDAR-derived terrain analysis with thematic provincial data, and supports consistent multi-theme atlas generation across multiple regions without rebuilding the workflow.
