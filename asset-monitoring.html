Project 01

Infrastructure Asset
Monitoring System

Designing a field-to-office GIS workflow to centralize infrastructure condition data across a remote First Nations community — replacing scattered spreadsheets with a real-time monitoring platform.

Domain Indigenous Infrastructure
Scale 100+ Assets
Role GIS Specialist
Status Deployed
Spatial Context // Asset Coverage
00
Context

Infrastructure assets in a remote Indigenous community — including buildings, water and wastewater systems, boardwalk roads, utilities, fuel storage, and electrical infrastructure — required regular condition assessments as part of a federal reporting program.

Assessments were conducted by visiting engineers on an infrequent basis. Between visits, there was no system in place to track asset conditions, changes, or maintenance status in a consistent or accessible way.

01
Problem

Asset data was distributed across engineering reports, spreadsheets, and institutional memory — with no centralized GIS database and no consistent workflow for tracking changes over time.

Core gap No single source of truth for asset condition data meant decision-makers couldn't access current status, prioritize interventions, or produce consistent federal reports without manually reconciling fragmented records.

The lack of spatial context also made it difficult to communicate asset status to non-technical stakeholders or to identify patterns across asset categories and locations.

02
Approach

The solution was structured around three components: standardized data collection, a centralized geodatabase, and a real-time visualization layer.

A Survey123 form was built to standardize field data collection by visiting engineers — capturing asset type, condition rating, notes, and photos in a structured format aligned to the ACRS reporting framework.

A geodatabase schema was designed with domains and subtypes to enforce data integrity and ensure consistency across asset categories. This schema became the single source of truth feeding all downstream reporting.

ArcGIS Dashboards with Arcade expressions were built on top of the live database — providing condition overviews, asset-level drill-downs, and visual summaries accessible to both field staff and program managers.

Outcome

Real-time condition visibility

The system established a centralized monitoring platform where all asset condition data flows from the field directly into a structured geodatabase, surfaced through dashboards that any stakeholder can access without GIS expertise.

The standardized schema also made federal reporting significantly faster — data no longer needed to be manually compiled from disparate sources before each submission cycle.

Transferable design The schema and workflow were designed to be replicable across similar communities participating in the same federal reporting program — not just a one-off solution.
